## What Makes This Different

Most student portfolios are a pile of isolated tutorial clones. Mine is deliberately built as **one connected system**:

- The **Job Search MCP Server** and **Autonomous Job Application Bot** were built to solve my *own* job hunt — dogfooded on real applications, not toy data.
- **VICTOR** isn't just a notebook — it ships with SHAP explainability and a live Streamlit dashboard, because a model nobody can interpret isn't production-ready.
- I'm currently closing out gaps intentionally: adding **YOLOv8 object detection**, a **LoRA/QLoRA fine-tuned LLM**, a **CLIP-based multimodal visual search system**, and a properly packaged **MLOps deployment**, so the portfolio covers CV, fine-tuning, retrieval, and deployment — not just one corner of ML.
- Two live startups (Agentlayer, Retix) mean the code has to survive contact with real users and real GST compliance rules, not just a grading rubric.
